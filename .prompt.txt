# 🧩 PHASE A — DATABASE (STRICT REQUIREMENTS)

## Task:

Write SQL queries for PostgreSQL (Supabase) to create:

---

### Users

* id (uuid primary key, default gen_random_uuid())
* name (text, required, min length 2)
* email (unique, required, valid email format)
* password (text, required, min length 6)
* role (text: 'admin' or 'member')
* created_at (timestamp default current time)

---

### Projects

* id (uuid primary key)
* name (text, required, min length 3)
* description (text)
* created_by (uuid → users.id)
* created_at (timestamp)

---

### ProjectMembers (TEAM MANAGEMENT — REQUIRED)

* id (uuid primary key)
* user_id (uuid → users.id)
* project_id (uuid → projects.id)
* role (text: 'admin' or 'member')
* added_at (timestamp)

Constraints:

* A user cannot be added twice to the same project (UNIQUE constraint)
* Cascade delete when project or user is deleted

---

### Tasks

* id (uuid primary key)
* title (text, required, min length 3)
* description (text)
* status (text: 'todo', 'in-progress', 'done', default 'todo')
* assigned_to (uuid → users.id)
* project_id (uuid → projects.id)
* due_date (timestamp)
* created_at (timestamp)

---

## Requirements:

* Add all foreign key constraints
* Add CHECK constraints for validation
* Add UNIQUE constraints where required
* Add indexes for performance (project_id, assigned_to)

---

## Validation (MANDATORY):

* Reject invalid emails (regex)
* Enforce password length
* Enforce allowed role values
* Enforce allowed status values
* Prevent duplicate project membership

---

## Verification:

* Insert test users
* Create project
* Add member to project
* Create task

👉 DO NOT MOVE FORWARD until DB works perfectly

---

# 🧩 PHASE B — BACKEND (NODE + EXPRESS)

## Setup:

* Express server
* PostgreSQL connection using `pg`
* Environment variables:

  * DATABASE_URL
  * JWT_SECRET
* Enable SSL for Supabase connection

---

## 🔐 Authentication:

Implement:

* POST /auth/signup
* POST /auth/login

Requirements:

* Hash password using bcrypt
* Generate JWT token
* Validate input before insertion
* Prevent duplicate email registration

---

## 🔒 Middleware:

### authenticateUser

* Verifies JWT
* Attaches user info (id, role)

### authorizeRoles(role)

* Restricts access based on role

---

## 📦 APIs:

### Projects

* POST /projects (admin only)
* GET /projects (only projects where user is a member)
* DELETE /projects/:id (admin only)

---

### Project Members (TEAM MANAGEMENT)

* POST /projects/:id/members (admin only)
* GET /projects/:id/members

---

### Tasks

* POST /tasks
* GET /tasks (only tasks from user's projects)
* PATCH /tasks/:id
* DELETE /tasks/:id

---

## ⚠️ Rules:

* Members can only update/delete tasks assigned to them
* Users can only access projects they belong to
* Tasks must belong to a valid project
* Assigned user must be part of the same project

---

## ✅ Validation:

Test using Postman:

* Signup/Login works
* Token authentication works
* Role restrictions enforced
* Project membership enforced
* Task assignment rules enforced

👉 Backend must be stable before frontend
